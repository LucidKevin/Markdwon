# 项目简介

## 一、项目目标

本项目旨在为数字货币相关业务提供一个多平台、自动化、可扩展的数据监控与告警系统，实现对主流交易所（如币安、火币、OKEX、Coinbase等）币种、合约等行情数据的采集、监控、分析和异常告警，辅助业务方及时发现市场异常、数据延迟、资金异动等风险。

## 二、项目价值

1. 自动化监控：自动采集并监控多平台币种、合约等行情数据，极大减少人工巡检成本。

2. 异常告警：支持多维度（如价格、涨跌幅、资金流向、数据延迟等）异常检测，及时通过WebChat等方式推送告警，提升响应速度。

3. 多数据源支持：支持多MySQL、Redis数据源，适应复杂业务场景。

4. 可扩展性强：平台化设计，便于后续扩展更多交易所、监控指标和告警方式。

5. 数据可视化：通过Web页面展示监控结果，便于运维和业务人员直观了解系统状态。

## 三、总体设计

### 1. 架构概览

- Spring Boot为基础框架，负责Web服务、定时任务、数据源管理等。

- 多数据源：支持aicoin、hw_coins等MySQL数据库，多个Redis实例。

- 定时任务：定时采集、监控、清理缓存。

- Web管理界面：基于Thymeleaf模板，展示监控数据。

- 消息推送：集成WebChat（企业微信/钉钉等）实现自动化告警。

- 容器化部署：支持Docker、Kubernetes部署。

### 2. 主要模块

- 数据采集与监控：PlatformTask、MonitorTask等定时任务，采集并监控行情数据。

- 数据存储与缓存：MySQL存储结构化数据，Redis缓存高频数据。

- 告警与推送：通过WebChatDao等实现异常告警推送。

- Web接口与页面：MonitorController等提供数据展示和接口服务。

# 核心功能

1. 多平台币种/合约行情采集

- 定时从各大交易所拉取币种、合约等行情数据，存入数据库和缓存。

2. 行情监控与异常检测

- 检测数据延迟、涨跌幅异常、资金流向异常等，支持自定义监控规则。

3. 自动告警推送

- 通过WebChat等方式，将异常信息推送给相关人员。

4. 缓存管理

- 定时清理不同粒度的缓存，保证数据实时性。

5. Web管理与数据可视化

- 提供Web页面展示监控结果、配置管理等。

## 五、主要接口（以MonitorController为例）

- /monitor/：监控首页，展示监控数据。

- /monitor/collect_url：采集URL展示，参数校验后返回采集结果页面。

- 其他接口（如测试、配置等）可根据业务需求扩展。
