# 热门榜项目

## 大额交易监控任务

**处理流程**：接收Kafka 消息 --> 调用配置服务获取交易对信息 --> 解析交易方向 -->保存到Mysql 数据库

**数据来源**

- Kafka Topic："dex_big_swap"

**配置服务**

- 接口地址：`http://eks-pro-bg-service-2e36358ea172309f.elb.ap-southeast-1.amazonaws.com:8787/index/getPairsConf`
- 参数：
  - pair: 流动性池地址
  - chain: 区块链网络

**数据存储**

- 数据库表：dex_big_swap

```java
- 主要字段：
  ```sql
  - pool_name: 交易对名称
  - pool_address: 流动性池地址
  - tx_address: 交易哈希
  - trader_address: 交易者地址
  - chain: 区块链网络
  - platform: 交易平台
  - token0_symbol: 代币0符号
  - token1_symbol: 代币1符号
  - token0_amount: 代币0数量
  - token1_amount: 代币1数量
  - amount_usd: 交易金额(USD)
  - trade_side: 交易方向(buy/sell)
  - swap_time: 交易时间
  - state: 状态
  - created_at: 创建时间
  - updated_at: 更新时间
  ```

## 热度趋势统计任务（话题）

**处理流程**：查询话题数据 --> 计算话题分数(点赞+1, 评论+2, 收藏+3) --> 将分数高于6分设置为热门 --> 存入Redis

**数据存储**

Redis 键

- trends_hot_version：存储当前热门版本
- trends_hot_all：<版本, HashMap<id, score>> 存储热门话题分数
- trends_hot_top10_version：存储前十的版本号
- Trends：<id, TopicModel> 话题详细信息

## 热度统计任务（交易对、币种、市场）

**处理流程:** 统计昨日交易对热度数据 --> 统计今日交易对热度数据 --> 统计交易对小时热度变化 --> 统计币种小时热度变化 --> 保存交易对、币种、市场日维度数据 --> 保存交易对、币种、市场小时维度数据 

**热度指标：**警告次数、自选添加次数、搜索频率、浏览次数
