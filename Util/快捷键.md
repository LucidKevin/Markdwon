```
taskkill /PID <PID号> /

```

```

```

```
jstack <PID>
```

# MacOS

## 根目录

**1. /Users**

​	这是用户的主目录，存放每个用户的个人文件和数据。每个用户都会有一个独立的文件夹（如 /Users/username），其中包含桌面、文档、下载、图片等子目录。这个目录通常是用户日常访问和使用的地方。

**2. /opt**

​	这是一个可选目录，用于存放第三方软件或工具，不是系统默认应用的一部分。开发者或系统管理员可能会在这里安装或存放一些自定义应用程序和包管理工具（如 Homebrew 的安装）。

**3. /private**

​	这个目录包含一些系统级的临时文件和日志文件，macOS 在运行时会利用这个目录中的文件来维持系统的状态，如 /private/tmp（临时文件）和 /private/var（变量文件）。平时用户不需要直接操作该目录。

**4. /System**

​	存放macOS操作系统的核心文件。这个目录中的内容包括系统框架、库、驱动程序等，系统需要依赖它来正常运行。普通用户通常不需要修改该目录下的内容，因为错误的操作可能会破坏系统。

**5. /usr**

​	包含用户级的可执行文件、库文件和系统工具。通常 /usr/bin 存放系统命令， /usr/sbin 存放系统管理员命令， /usr/lib 存放系统库。这个目录是macOS系统的核心组成部分。

**6. /Library**

​	包含系统和应用程序使用的资源文件、支持文件、配置文件和共享库。这个目录与 /System/Library 不同，它更侧重于为应用程序和系统扩展提供支持，并且通常由用户或应用程序使用。

**7. /bin**

​	存放常用的系统命令，如 ls, cp, mv, bash 等。这些命令对所有用户可用，提供基本的命令行工具和功能。/bin 是系统启动时必须的目录之一。

**8. /sbin**

​	这个目录类似于 /bin，但它包含的是系统管理员（root 用户）才常用的工具和命令，如 ifconfig, shutdown, reboot。普通用户一般没有权限使用其中的命令。

## Users

1. Library

​	•Application Support：存放用户安装的应用程序的支持文件和数据。

​	•Preferences：存放用户的应用程序设置和偏好。

​	•Caches：存放用户的缓存文件。

​	•Logs：用户应用程序生成的日志文件。

​	•Containers：存放用户的应用程序容器数据，例如沙盒应用程序的数据。

2. .zshrc：包含用户的 shell 配置和环境设置。

## usr

1. local：通常用于存放用户安装的软件和其他非系统默认的文件。这个目录被设计为系统的“本地”目录，专门用于存放那些不随系统默认安装的程序和文件。
2. bin：存放用户安装的可执行文件和脚本。将自定义的命令行工具放在这里，可以使它们在系统的 PATH 环境变量中可用，便于从终端直接调用。
3. sbin：存放系统管理员（root 用户）安装的可执行文件和脚本，类似于 bin 目录，但通常用于系统管理和维护任务
4. lib：存放用户安装的库文件。软件在运行时需要这些库文件来提供功能和支持。
5. share：存放用户安装的共享数据文件和文档，如手册、帮助文件和其他程序所需的共享资源。
6. Include：存放用户安装的头文件（通常是 C/C++ 的头文件），这些头文件提供编译时所需的接口和声明。
